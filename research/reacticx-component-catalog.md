# Reacticx Component Catalog

This catalog covers all 127 components shipped under `apps/app/app/components/reacticx/`. Each entry summarises what the component does, its most important props, what runtime risk it carries against our current Expo + Reanimated + Skia stack, and a plausible landing spot inside a wearable/health app. Entries are derived from each component's index file and accompanying `types.ts` / `*.types.ts` / `*.props.ts`. Pre-audited components (`energy-orb`, `elastic-slider`, `disclosure-group`, `picker`, `dialog`) are re-included here in the standard structure.

> **How to read this catalog**
>
> - **safe** — plain RN + Reanimated only; no worklets-per-frame, no Skia, no missing deps, no NativeWind required.
> - **worklet-frame-cb** — uses `useFrameCallback` (runs every frame on the UI thread; budget watch).
> - **worklet-animated-blur** — uses `Animated.createAnimatedComponent(BlurView)` with `useAnimatedProps` (per-frame worklet -> bridge pressure on the blur layer).
> - **skia-static** — uses `@shopify/react-native-skia` but no `useFrameCallback` (a Skia canvas paints, but no per-frame work driven from JS).
> - **needs-nativewind** — uses literal `className="…"` strings (requires NativeWind, which we do not have configured).
> - **needs-extra-dep** — imports a module that is not in our `package.json` (e.g. `svg-path-properties`, `@sbaiahmed1/react-native-blur`, `@expo/vector-icons`, `react-native-easing-gradient`, `react-native-gifted-chat`, `react-native-qrcode-styled`, `expo-symbols`, `expo-status-bar`, `expo-video`, `@lodev09/react-native-true-sheet`, `@hugeicons/*`, `@react-navigation/elements`) or pulls from a broken intra-reacticx path like `@/components/molecules/Loaders/circular`.
>
> A component can technically hit more than one bucket (e.g. Skia + frame callback). The risk tag picked is the *most blocking* one for adoption, in the order shown above.
